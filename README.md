# Проект. Обучение модели детекции на основе сгенерированной разметки
Реальность работы специалиста по Data Science бывает очень суровой. Например, иногда нет ни данных, ни разметки, а сроки горят. В этом проекте вам предстоит «пощупать» реальность — решить хардкорный кейс с похожими ограничениями. Вам нужно обучить модель на задачу детектирования шлемов на людях, при этом у вас есть только датасет с картинками, а разметки нет.

## Общее описание проекта
Бизнес-контекст. На промышленных предприятиях, стройках и складах важно соблюдать технику безопасности. Одно из главных правил — носить защитный шлем. Важно вовремя заметить нарушение (сотрудника без шлема), потому что несоблюдение правила повышает риск травм, инцидентов, простоев и штрафов. Контроль по камерам вручную сложно и дорого, компании отдают предпочтение системам, которые автоматически находят людей без шлемов. В проекте вы построите модель для такой системы. 
План проекта. Вам нужно:
* Выбрать модель детекции. Получить для неё автоматическую разметку датасета с помощью VQA-модели, *используя фреймворк vLLM. Оценить её качество.
* Обучить модель детекции на получившемся датасете и предложить пути для улучшения метрик.
Датасет. Скачайте датасет (251 Мб) с изображениями людей в шлемах и без них (https://code.s3.yandex.net/deep-learning-cv/images.zip?etag=c508a8c704db94187fe6e0b8219479e9-4). Датасет содержит только картинки. Не обогащайте датасет внешней информацией.

## Этап 1. Генерация разметки
* Работа с данными
Загрузите датасет и визуализируйте часть данных. Посмотрите на несколько примеров и оцените потенциальные проблемы при обучении на таком датасете: в каких случаях и как сильно может ошибиться VQA-модель. Выводы сохраните в Markdown-ячейке.
* Генерация разметки
    - Выберите VQA-модель для генерации разметки.
    - Используя примеры из датасета, сформулируйте промпт для генерации.
    - Запустите выбранную модель в vLLM на ВМ.
    - Извлеките из генераций разметку и приведите её к нужному формату для выбранной модели детектора.
    - Оцените качество полученной разметки.
Результат: выводы о возможных проблемах, разметка для обучения моделей детекции. 

## Этап 2. Обучение модели детекции шлемов на людях
Обучение модели
Получите модель, которая сможет выделять на изображениях людей, на которых нет шлема. Целевая метрика — mAP. По результатам обучения должно быть очевидно, что модель сходится (уменьшение loss, рост mAP и т. д.).
Анализ результатов
Оцените общее качество работы модели и рассмотрите примеры с ошибками.
Предложите способы улучшения метрик детектирования.
Результат: дообученная модель детекции, анализ результатов. 

## Оформление проекта
Оформите проект:
Вывод картинок из датасета выполняйте в Jupyter Notebook'е, реализацию всех функций и обучения выполняйте в том же ноутбуке.
Код ноутбука должен быть разделён на смысловые блоки и легко читаем.
Все ячейки должны выполняться по порядку.
Выводы ноутбука должны быть сохранены в отправке на проверку.
В качестве результата предоставьте Jupyter Notebook с кодом и визуализацией выполненных этапов работ.
